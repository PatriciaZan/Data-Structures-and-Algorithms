Big O Notation

helps us understanf how long an algorithm will take to run(time complexity) or how much memory it will need as the amount of data it handles grows(space complexity)

O(n) -> execution time grows linearly in proportion to the size of the input data(n)

O(1) -> constant time, execution time remais constant regardless of the input size

O(n^2) -> execution time grows quadratically with the size of the input data(expensive)

O(log n) -> grow logarithmically with the size of the input. as the input size grows the time it takes for the algorithm to run encreases slowly
